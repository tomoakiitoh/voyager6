# voyager6 拡張プラン2: ハイアマチュア望遠鏡モード

PLAN.md(Phase 0〜5、実装済み)の続編。対象を二層化する:
**入口=肉眼観望(現状の全天図、子どもも大人も同じ画面)、奥=望遠鏡ユーザー向け(本プラン)**。
既存の見た目・軽さは壊さず、「望遠鏡モード」への切替で深い機能を開く。
子ども向けの別モードは作らない(F6の設計方針参照)。

## 0. アーキテクチャ方針の変更

- **単一HTML制約を撤廃**する。静的サイト(GitHub Pagesのまま、サーバ不要)+遅延ロードのデータファイル群へ移行
- データ生成は既存のActionsパイプラインを二系統に分ける:
  - **build時(push)**: 恒星カタログ・DSO・星座線など静的データ
  - **schedule時(cron、週2回程度)**: 彗星・小惑星の軌道要素(MPC最新値)。取得→検証→JSON生成→自動デプロイ。取得失敗時は前回データを維持してデプロイ(サイトを壊さない)
- テスト必須ゲート(テスト失敗=デプロイなし)は両系統で維持

## F1. ズーム+望遠鏡モード(最優先 — 深い星表の前提)

- ピンチ/ホイール/ダブルタップでズーム、ドラッグでパン。現在の全天図はズーム最小状態として連続的につながる
- 表示反転切替: 正立(肉眼・ファインダー)/ 鏡像(天頂ミラー)/ 倒立(ニュートン直焦点)
- **FOVサークル**: Telrad(0.5/2/4°)、ファインダー(任意°)、アイピース(望遠鏡焦点距離+アイピース焦点距離+見かけ視界から実視界を計算)。機材プリセットをlocalStorage相当(※Pages上なので普通にlocalStorageでよい)に保存
- 赤道座標グリッド表示、「この位置のRA/Dec」表示、RA/Dec指定で視野中心をジャンプ
- 完了条件: M31を視野中心に置き、Telradサークルと8等星のパターンで実際に星飛びできるチャートが印刷なしで使えること

## F2. 深い星表(V≤10、約35万星)

- ソース: **AT-HYG**(Tycho-2全量+Gaia DR3統合、CC BY-SA系。build_dataで最新版CSVを取得)
  - リポジトリ: astronexus/athyg (Codeberg/GitHubミラー)
- 抽出: V≤10.0 → 約35万星。フォーマットはバイナリ(RA/Dec各Float32、mag/B-V各Uint8量子化)≈3.5MB、gzip後2MB台
- ロード戦略: 全天表示時は既存の埋め込み5等データ(初期表示の速さを守る)。望遠鏡モードON時に深層カタログをfetch。等級層(≤7 / ≤8.5 / ≤10)×空域タイル(簡易HEALPix or RA/Dec格子)に分割し、視野と倍率に応じてロード
- 描画: 可視領域カリング+視野角に応じた限界等級カット。Canvas 2Dのまま(視野内数千星なら余裕)。厳しければOffscreenCanvas/WebGL化はF6以降の課題に
- **歳差を真面目に**: J2000→現在date(IAU 2006簡約式)。10等チャートでは必須。固有運動はAT-HYGに含まれるので大きい星だけ適用(μ>0.2″/yr等)
- 完了条件: 実際の望遠鏡視野(または既存星図ソフト)と8〜10等星の並びが一致すること

## F3. 彗星

- データ: MPCの現行彗星軌道要素(CometEls.txt / Soft00Cmt.txt形式)をcronジョブで取得→JSON化
- 計算: 軌道要素からの位置計算を離心率で分岐(e<1楕円=ケプラー方程式、e≈1放物線=Barker、e>1双曲線)。汎用変数法で一本化してもよい
- 光度モデル: m = M1 + 5 log₁₀Δ + 2.5·K1·log₁₀r。表示閾値は既定V≤12(設定で変更可)
- 表示: 彗星マーカー+名前+現在等級。尾の向き(反太陽方向)の短い矢印は余裕があれば
- 完了条件: 表示中の彗星1つをJPL Horizonsと突き合わせ、位置誤差が分角オーダーであること

## F4. 小惑星

- MPCORB全量(140万)は配らない。cronジョブ内で全numbered天体の現在等級を概算し、**「現在V≤11の小惑星」だけ軌道要素JSONで配信**(通常数十〜数百個)。位置はクライアントで都度計算
- 四大小惑星(Ceres/Pallas/Juno/Vesta)は明るさによらず常時掲載
- 完了条件: Vestaの位置をHorizonsと照合

## F5. DSO(星雲・星団・銀河)

- Messier 110 + **OpenNGC**からNGC/ICをV≤12程度で抽出(数千件)。種別アイコン(銀河=楕円、散開=破線円、球状=十字円、星雲=方形)+長径楕円+向き
- ラベルは倍率連動で間引き。検索ボックス(M31 / NGC253 / 名称)から視野ジャンプ
- ライセンス: OpenNGC=CC-BY-SA-4.0、出典表記をデータ先頭とaboutページに

## F6. 夏休みコンテンツ(並行して小さく)

**設計方針: 「子ども向けモード」は作らない。** 小学生以降の子どもが欲しいのは
本物の道具であって、ひらがな化した別バージョンではない(スマホとキッズケータイの関係)。
子どもへの提供価値は「本物が無償・ログイン不要・ブラウザだけで動く」こと自体
(学校配布Chromebookでも使える)。用語も大人と同じものを使う。

- ペルセウス座流星群ページ(2026年は極大8/13+新月の当たり年、見頃8/12夜〜13明け方)— 内容は大人と同じ、8月上旬までに
- 観察記録シート(印刷用1枚) — 子ども用ワークシートではなく、実際の観測者が使う観測ログ様式(日時・場所・空の状態・スケッチ欄・メモ)をそのまま渡す。自由研究にはそれが一番効く

## F7. 方位磁針(コンパス)表示 — スマホ内蔵センサ(後日)

観望中に「今どっちを向いているか」を知るための方位表示。要望: スマホの磁力・傾きセンサを読む。

- **Web で可能**: `DeviceOrientationEvent`。iOS Safari は `webkitCompassHeading`(真方位に近い)、
  Android Chrome は `deviceorientationabsolute` の alpha で絶対方位が取れる。要 HTTPS(充足済み)。
  **iOS 13+ は要ユーザー操作での `DeviceOrientationEvent.requestPermission()`**(ボタンで許可を求める)。
- 段階案:
  1. **方位ロゼット/ヘディング数値**の表示(まずはこれ。全天図・望遠鏡モードとも)。キャリブレーション注意書き添え。
  2. 望遠鏡モードで**端末の向きに視野を追従**(AR的「空にかざす」)。傾き(beta/gamma)込みで視野中心を端末の指す方向へ。ジャイロのドリフト・磁気の乱れ対策(平滑化・手動リセット)が要る。
- 精度は環境依存(近くの金属・磁気)。あくまで補助表示とし、正確な導入は星の並びで行う前提。
- **Web で実用精度が出なければアプリ化**(PWA/Capacitor でセンサ取得を安定化)を検討。そのタイミングでよい。
- 実装順は任意(F3〜F5 と独立)。センサ許可 UI は他機能に干渉しないよう小さく。

## 実装順の提案

**F1 → F2 → F3 → F4 → F5**(F6・F7は随時割り込み可)。
F1が先なのは、ズームと反転がないと10等星表は表示できても使えないため。F3/F4はcron基盤を共有するので連続で。

## 検証(全体)

1. 彗星・小惑星はJPL Horizonsとの突き合わせテストをCIに組み込む(代表天体・固定日時のスナップショットテスト)
2. 歳差適用後の恒星位置は既存星図ソフトの表示と照合
3. 実地検証は所有者(天文専門家)が行う — 望遠鏡モードの完了判定は実視野との一致
