# 天文情報サイト 実装プラン

[PLAN.md](PLAN.md) で作った単一HTMLの星座早見を、天文情報サイトのトップに据えて拡張する。
PLAN.md は「早見盤そのもの」の仕様として引き続き有効。この文書はサイト化の分だけを扱う。

---

## 1. ゴール

日本で日常的に使える天文情報サイト。「今夜どこに何が見えるか」が地点を選ぶだけで分かることを最優先にする。

- 対象ユーザー: 自分・家族 + 同じものを探している国内の観望者
- 方針: **静的サイト**。サーバー不要、外部API不要、表示が速い
- 公開: GitHub Pages
- 差別化: 早見盤が主役。既存サイトは「情報の一覧」止まりで、空の絵と情報が繋がっていない

## 2. サイト構成

| ページ | 内容 |
|---|---|
| `/` | **星座早見**(現行 index.html)。トップに全画面で置く |
| `/tonight/` | **今夜の空**。地点別に 日の出入り・薄明・月の出入り・月相・惑星の見え方 |
| `/calendar/` | **天文現象カレンダー**。月相・惑星の現象・流星群・食 |
| `/credits/` | **出典とライセンス**。CC BY-SA の表示義務を果たす |

各ページから早見盤へ「その時刻の空を見る」リンクを張る(`/?t=2026-08-12T22:00&lat=...`)。
情報 → 空の絵 の往復ができることがこのサイトの核。

## 3. ライセンス上の制約 (公開に伴い発生)

個人利用から**再配布**に変わるため、次を必ず満たす。

- **HYG Database v3.8 = CC BY-SA 2.5**
  - 出典表示(著作者・ライセンス名・リンク)を `/credits/` と全ページのフッターに置く
  - 星表を加工した配布物(`data.js` の恒星部分)は **同じ CC BY-SA 2.5 で提供**する。
    `data.js` の先頭コメントに明記し、`/credits/` からも辿れるようにする
- **d3-celestial = BSD-3-Clause** — 著作権表示とライセンス全文を `/credits/` に載せる
- **自作コード(計算・描画・サイト)は MIT** とし、データとライセンスを分けて明示する

サイトのコードとデータでライセンスが違うので、「どのファイルがどれか」を credits に表で書く。

## 4. リポジトリ構成 (再編)

早見盤は単一HTMLのままだが、計算エンジンを 3 ページで共有するので部品に切り出す。
ビルドは既存 `tools/*.py` の延長 (npm 依存なし)。

```
planisphere/
├── PLAN.md / SITE_PLAN.md
├── src/
│   ├── astro.js          ← 計算エンジン (index.html の ASTRO を切り出し。全ページ共有)
│   ├── render.js         ← 早見盤の描画 (RENDER を切り出し)
│   ├── sky.js            ← 出没・薄明・現象計算 (新規。今夜の空 / カレンダーが使う)
│   ├── sites.js          ← 観測地プリセット (都道府県庁所在地 + 主要観測地)
│   ├── data.js           ← 星表 (build_data.py が生成)
│   ├── events.js         ← 流星群・食のテーブル (build_events.py が生成)
│   ├── style.css         ← 共通スタイル
│   ├── layout.html       ← 共通ヘッダ・フッター・<head> テンプレート
│   └── pages/
│       ├── index.html    ← 早見盤 (UI 部分のみ。astro/render は build が差し込む)
│       ├── tonight.html
│       ├── calendar.html
│       └── credits.html
├── tools/
│   ├── build_data.py     ← (既存) 星表生成
│   ├── build_events.py   ← (新規) 流星群・食テーブル生成
│   └── build_site.py     ← (新規) src/ を結合して dist/ を出力
├── dist/                 ← 公開物 (GitHub Pages が配信)
└── tests/
```

- **共有 JS は `<script src>` で読む**。`file://` でも動く (fetch を使わないため)。
  ページを跨いでブラウザキャッシュが効くので、単一HTMLに全部埋めるより速い
- テストは `src/*.js` を直接読む (現行の「HTML から切り出す」方式より単純になる)

## 5. 天文計算の追加分 (`src/sky.js`)

現行エンジン(太陽 0.3°・月/惑星 0.5° 精度)を土台にする。到達できる精度は正直に書く。

1. **出没・薄明時刻**: 高度が閾値を横切る瞬間を 1分刻みスキャン + 二分法で求める
   - 太陽の出入り: 高度 −0.833°(大気差 + 視半径)
   - 市民/航海/天文薄明: −6° / −12° / −18°
   - 月の出入り: 高度 +0.125°(大気差 − 地平視差 + 視半径)
   - **精度**: 太陽は誤差 1分以内。月は位置精度 0.5° ぶんが効いて **誤差 2〜3分**。
     表示は「分」までとし、秒は出さない
2. **月相**: 輝面比・月齢・朔望(新月/満月の瞬間は輝面比の極値を二分法で)
3. **惑星の現象**: 黄経差・離角を 1日刻みでスキャンし極値・零点を検出
   - 合(内合/外合)・衝・最大離角 → 日付精度 ±1日程度で出せる
   - **留は出さない**: 視赤経の停留は現行精度では日付が数日ぶれるため
4. **観望好機**: その夜の「天文薄明終了 〜 開始」の間で、惑星が高度20°以上にある時間帯

## 6. データの追加分

| データ | 内容 | ソース | 生成 |
|---|---|---|---|
| 流星群 | 主要12群の極大日時・ZHR・輻射点 | IMO の年次カレンダー(手書きテーブル) | `build_events.py` |
| 日食・月食 | 2026〜2035年の日付・種類・日本での見え方 | NASA Eclipse Web Site の五千年カタログ(手書きテーブル) | `build_events.py` |
| 観測地 | 都道府県庁所在地47 + 主要観測地(富士山・八ヶ岳ほか) | 手書き | `sites.js` |
| メシエ天体 | 110天体の座標・等級・日本語名 | 手書き/公開表 | `build_data.py` に追加 |

**食を計算しない理由**: 地心の最大食時刻は Meeus の方法で出せるが、「東京での食分と接触時刻」には
ベッセル要素が必要で、現行の精度では出せない。**確定済みの表を持つほうが正確で軽い**。
流星群の極大日時も同様(軌道計算ではなく観測に基づく予報値)。
ただし **その年の月明かり条件(極大夜の月相・月の出入り)は計算で自動生成する** — ここが手作業の表にはない価値。

## 7. 実装フェーズ

各フェーズ完了ごとにブラウザで目視確認 + テストを通してから次へ進む。

### Phase A: リポジトリ再編とビルド (挙動は変えない)
- `index.html` を `src/astro.js` / `src/render.js` / `src/pages/index.html` に分割
- `tools/build_site.py`: `src/` を結合して `dist/` を出力。`layout.html` で共通ヘッダ/フッターを被せる
- テストを `src/*.js` 直読みに変更
- **完了条件**: `dist/index.html` が現行と同じ空を描く。テスト27件が通ったまま

### Phase B: 共通レイアウトと出典ページ
- ヘッダ(サイト名・ナビ)、フッター(クレジット)、`/credits/`
- CC BY-SA 2.5 / BSD-3-Clause / MIT の表示、どのファイルがどれかの表
- **完了条件**: 全ページからライセンス表示に到達できる。3章の義務を満たしている

### Phase C: 今夜の空
- `src/sky.js`(出没・薄明・月相)、地点プリセット + localStorage 保存
- 日の入り〜日の出の時系列、惑星の観望好機、月相アイコン
- 早見盤への「この時刻の空を見る」リンク
- **完了条件(検証)**: 東京・大阪・札幌の日の出入り/月の出入りを海上保安庁の暦と突き合わせ、
  太陽は誤差1分以内、月は3分以内。テストに固定値で残す

### Phase D: 天文現象カレンダー
- 月ごとの表示。月相・惑星の現象(計算)・流星群・食(テーブル)
- 流星群には**その年の月明かり条件**を自動で添える
- **完了条件(検証)**: 2026年の衝・合・最大離角の日付を天文年鑑/Horizons と突き合わせ、±1日以内

### Phase E: 早見盤の強化
- メシエ天体の表示トグル、地点プリセット連携、URL パラメータ(`?t=&lat=&lon=`)対応
- **完了条件**: `/tonight/` から時刻付きリンクで飛ぶと、その時刻の空が開く

### Phase F: 公開
- GitHub Pages へのデプロイ(GitHub Actions で `build_site.py` を回して `dist/` を公開)
- `<title>`/description、OGP 画像、`sitemap.xml`、favicon
- **完了条件**: 公開URLで全ページが表示され、スマホで実用になる
