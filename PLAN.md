# Web星座早見 実装プラン

Claude Code に実装を依頼するためのプラン文書。このファイルをプロジェクトルートに置き、「PLAN.md に従って Phase 1 から実装してください」と指示する想定。

---

## 1. ゴール

ブラウザだけで動く星座早見。指定した日時・場所の空を、**真円形の全天図**(天頂が中心、地平線が外周)として描画する。自分・家族の観望用なので、見やすさと動作の軽さを優先し、学術的精度は「星座早見盤として十分」なレベルでよい。

- 対象ユーザー: 自分・家族(観望のお供)
- 対応環境: PC・スマホのモダンブラウザ(Chrome / Safari)。オフラインでも開けること
- 配布形態: **単一 HTML ファイル**(データも JS に埋め込み、外部依存なし)

## 2. 画面仕様

### 2.1 全天図(メインビュー)

- 真円の全天図。**天頂を中心、高度0°の地平線を外周**とする方位図法
- 投影は **正距方位図法**(半径 r ∝ 90° − 高度)。実装が最も単純で歪みの性質も直感的
- 円の外周に方位ラベル **北・東・南・西**。⚠️ 星図は「見上げた空」なので地図と鏡像:**北を上にしたとき東は左**。ここを間違えると全部破綻するので Phase 1 の検証項目にする
- 背景は濃紺〜黒。円の外側はさらに暗くして盤面らしく
- 補助線(トグル可): 高度30°/60°の同心円、方位線、天の赤道、黄道

### 2.2 UI(盤面の外)

- 日付ピッカー + 時刻スライダー(0:00–24:00、分単位)
- 「**今の空**」ボタン: 現在日時にリセット
- 時間送り: ±1日 / ±1時間 ステッパー、▶ アニメーション再生(60倍速・3600倍速切替)。日周運動・年周運動を眺められるのが Web 化の最大のうまみ
- 表示トグル: 星座線 / 星座名 / 星名 / 惑星・月 / 天の川 / 補助線
- 観測地: 既定は **東京(35.68°N, 139.77°E, JST固定)**。設定パネルで緯度経度を手入力できれば十分(geolocation は任意、なくてもよい)
- 星・惑星をタップ/クリック → 名前・等級・高度方位をポップアップ表示

### 2.3 スマホ配慮

- Canvas は devicePixelRatio 対応で高精細に
- 縦持ちで盤面が正方形に収まるレイアウト。コントロールは下部に集約
- 観望時に眩しくないよう、UI も暗色。おまけで「赤色ナイトモード」トグルがあると実際の観望で嬉しい(CSS フィルタで全体を赤くするだけで可)

## 3. 天文計算仕様

すべて自前実装(ライブラリ不使用)。精度目標: 恒星の位置ずれ < 0.2°、惑星 < 0.5°、月 < 0.5°。星座早見用途には十分。

1. **時刻系**: JST → UTC → ユリウス日 JD。ΔT は無視してよい
2. **恒星時**: GMST(IAU 1982 近似式)→ 地方恒星時 LST = GMST + 経度
3. **座標変換**: 赤道座標 (RA, Dec) → 時角 H = LST − RA → 地平座標 (高度 alt, 方位 az)。標準的な球面三角の式
4. **大気差**: 省略(地平線際の見え方が少し変わるだけ)
5. **歳差**: 省略可(J2000 元期のままで2050年頃まで誤差 < 0.5°)。余裕があれば簡易補正
6. **太陽**: 平均軌道要素からの低精度式(Meeus 簡約版)。薄明判定と黄道表示に使用
7. **惑星(水〜土)**: Paul Schlyter "How to compute planetary positions" の方式(ケプラー軌道要素+主要摂動)。精度は分角オーダーで十分
8. **月**: 同じく Schlyter 方式(主要摂動項入り、誤差 ~2′)。月齢は太陽との離角から算出し、満ち欠け形状をアイコン描画

## 4. データ仕様

単一 HTML に `const STARS = [...]` 形式で埋め込む。データ生成は別途ビルドスクリプト(Python)で行い、生成結果を HTML に貼り込む。**ビルドスクリプトもリポジトリに残す**こと(等級上限を変えたくなったとき再生成できるように)。

| データ | 内容 | ソース | 目安サイズ |
|---|---|---|---|
| 恒星 | 等級 ≤ 5.0(約1,600星)。[RA, Dec, 等級, B−V] を小数2〜3桁に丸めた配列 | HYG Database v3(CC BY-SA 2.5) | 〜50 KB |
| 星名 | 1.5等より明るい星+主要星約60個の日本語名(ベガ、シリウス、アンタレス…) | 手書きテーブル | 〜3 KB |
| 星座線 | 88星座の線分(HIP 番号ペア → RA/Dec に解決して埋め込み) | Stellarium skyculture "modern" または d3-celestial のデータ | 〜30 KB |
| 星座名 | 88星座の日本語名(こと座、さそり座…)+ラベル位置(星座の重心) | 手書きテーブル+計算 | 〜4 KB |
| 天の川 | 等高線ポリゴン(簡略化して頂点を間引く) | d3-celestial の mw.json を簡略化 | 〜20 KB |

合計 ~110 KB 程度 → 単一 HTML でまったく問題なし。

ライセンス注意: HYG は CC BY-SA(個人利用は問題なし、HTML 内コメントに出典明記)。d3-celestial のデータは BSD 系。出典コメントを埋め込みデータの先頭に付けること。

## 5. 描画仕様(Canvas 2D)

描画順(下→上): 背景円 → 天の川 → 補助線 → 星座線 → 恒星 → 星座名 → 惑星・月 → 方位ラベル

- **恒星**: 半径 = f(等級)。例: r = max(0.5, 4.5 − 0.75 × mag) px(要調整)。1等星以上はわずかなグロー(radialGradient)。B−V から色味を薄く付ける(青白 ↔ オレンジ)
- **星座線**: 細線、半透明の青灰色。地平線の下の部分は描かない(線分単位でクリップ)
- **星座名**: 小さめのフォント、低コントラスト。混雑時は高度が低い星座から間引き
- **惑星**: 恒星と区別できる色付きマーカー+名前(金星=白金、火星=赤、木星=クリーム、土星=黄)。金星・木星は明るさ相応に大きく
- **月**: 月齢に応じた欠け形状を 10px 程度で描画
- **天の川**: 白 3〜5% の半透明塗り重ね
- 毎フレーム全再描画で OK(星1,600個 + 線なら 60fps 余裕)。アニメーション時のみ requestAnimationFrame

## 6. ファイル構成

```
planisphere/
├── PLAN.md              ← このファイル
├── index.html           ← 完成品(単一ファイル、データ埋め込み済み)
├── tools/
│   └── build_data.py    ← HYG等をDL・変換し data.js 断片を出力
└── tools/output/        ← 生成された data 断片(index.html に貼り込む)
```

開発中は index.html 内をセクションコメントで区切る:
`<!-- DATA -->` `<!-- ASTRO(計算) -->` `<!-- RENDER(描画) -->` `<!-- UI -->`

## 7. 実装フェーズ

各フェーズ完了ごとにブラウザで目視確認してから次へ進む。

### Phase 0: データ準備
- `tools/build_data.py`: HYG v3 CSV をダウンロード → 等級≤5.0 を抽出 → JS 配列断片を出力
- 星座線・天の川データも同様に変換
- 完了条件: 生成された data 断片のサイズと星数をログで確認

### Phase 1: 骨格+恒星描画
- HTML/Canvas 骨格、正距方位投影、JD/GMST/座標変換、恒星のプロット、方位ラベル
- 固定日時(例: 2026-07-13 21:00 JST 東京)で描画
- **完了条件(検証)**: 7月中旬21時の東京で「夏の大三角(ベガ・デネブ・アルタイル)が東の空高く」「さそり座アンタレスが南の低空」「北斗七星が北西」に見えること。東西の鏡像が正しいこと

### Phase 2: 星座線・星座名・星名
- 星座線のクリップ描画、日本語星座名ラベル、主要星名
- 完了条件: 主要星座が一目で判別でき、ラベルが重なりすぎないこと

### Phase 3: 時刻操作 UI
- 日付・時刻コントロール、「今の空」、ステッパー、アニメーション再生
- 完了条件: 時刻送りで星が東→南→西に日周運動すること。日付送りで同時刻の空が約1°/日 西にずれること

### Phase 4: 太陽・月・惑星
- 太陽(薄明の背景色変化: 太陽高度 > −6° で紺を明るく)、月(位置+満ち欠け)、水〜土星
- **完了条件(検証)**: 実装日当日の惑星位置を国立天文台「今日のほしぞら」と見比べて向き・高度が合うこと
- 月・惑星のタップで名前・高度方位表示

### Phase 5: 仕上げ
- 天の川描画、補助線トグル、赤色ナイトモード、スマホレイアウト調整
- 完了条件: スマホ実機(縦持ち)で操作でき、オフライン(file:// 直開き)で動くこと

## 8. 検証方法(全体)

1. 国立天文台「今日のほしぞら」(https://eco.mtk.nao.ac.jp/koyomi/skymap/)と同日時・東京で比較
2. 既知イベントの再現: 冬(2月20時)にオリオン座が南中、夏(8月21時)にさそり座が南中
3. 極端ケース: 高緯度(緯度65°を手入力)で周極星の挙動、日付変更線をまたぐ時刻送り
